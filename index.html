75 . Sort Colors

发表于 2018-02-07 | 分类于 Leetcode

75 . Sort Colors

简单思考后发现此题不难,给出如下解法: 

void sortColors(int[] nums) {
    int red = 0;
    int blue = 0;
    int white = 0;
    for (int i : nums) {
        switch (i) {
            case 0:
                red++;
                break;
            case 1:
                white++;
                break;
            case 2:
                blue++;
                break;
        }
    }
    int[] result = new int[red + blue + white];
    for(int i = 0; i < red; i++) {
        result[i] = 0;
    }
    for (int i = red; i < red + white; i++) {
        result[i] = 1;
    }
    for (int i = red + white; i < red + white + blue; i++) {
        result[i] = 2;
    }
    for(int i = 0; i < result.length; i++) {
        nums[i] = result[i];
    }
}

ide提示那段数组复制的代码可以用System.arraycopy()方法来实现.

不过我又想了下,费这大劲搞个result是为了啥…

于是改进版:

用这段代码来测试之后:

时间为2624ms.

用来测试之前的那段代码的话,会堆溢出.

我看到另一种解法:

就是遍历过去,1不动,0往左换,2往右换.

有一个小细节就是后面的++是运算完了再执行的.

测试之后时间是2642ms.

时间上并没有占到优势.

不过后一种的空间复杂度为o(n)

而我写的空间复杂度为o(n+m)
